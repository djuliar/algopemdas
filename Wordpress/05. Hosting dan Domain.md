# Hosting dan Domain

## 1. Pengertian dasar

**Domain**
Nama unik yang dipakai untuk mengidentifikasi situs di internet (mis. `example.com`). Domain menggantikan alamat IP yang sulit dihafal. Domain terdiri dari label (nama) dan TLD (Top-Level Domain) seperti `.com`, `.id`, `.org`.

**Hosting (Web Hosting)**
Tempat/layanan penyimpanan file, database, dan layanan lain agar situs dapat diakses lewat internet. Hosting menyediakan server, koneksi jaringan, dan layanan pendukung (FTP, database, email, kontrol panel).

---

## 2. Struktur domain & istilah penting

* **TLD (Top-Level Domain)**: `.com`, `.net`, `.org`, `.id`, `.co.id` — bisa global (gTLD) atau negara (ccTLD).
* **Second-level domain**: bagian langsung sebelum TLD (`example` di `example.com`).
* **Subdomain**: `blog.example.com` — bagian tambahan di depan domain utama.
* **Registrar**: penyedia jasa pendaftaran domain (mendaftarkan nama & menyimpan data WHOIS).
* **WHOIS**: database yang menyimpan data pemilik domain (bisa privat).
* **Nameserver**: server DNS yang mengatur mapping domain → IP. Biasanya diberikan oleh provider hosting atau DNS provider.
* **DNS (Domain Name System)**: sistem yang menerjemahkan nama domain ke alamat IP dan mengatur record (A, CNAME, MX, TXT, dsb).

---

## 3. Jenis hosting (keunggulan & kegunaan)

* **Shared Hosting**

  * Banyak website di satu server. Murah, cocok untuk situs kecil/baru (blog, company profile).
  * Kekurangan: sumber daya dibagi → performa bisa turun jika neighbor padat.

* **VPS (Virtual Private Server)**

  * Server virtual dengan resource terisolasi. Lebih kuat & fleksibel (akses root), cocok untuk traffic medium, aplikasi custom.
  * Perlu sedikit administrasi (jika unmanaged).

* **Dedicated Server**

  * Server fisik penuh untuk Anda. Sangat powerful, untuk trafik besar atau kebutuhan khusus. Mahal & perlu manajemen.

* **Cloud Hosting**

  * Layanan terdistribusi (scale mudah), bayar sesuai penggunaan. Baik untuk skala dinamis. Contoh: auto-scaling, layanan terkelola.

* **Managed Hosting / Platform-as-a-Service (PaaS)**

  * Provider manage infrastruktur (backup, security, updates). Cocok jika ingin fokus pada aplikasi saja.

* **Reseller Hosting**

  * Menyewakan kembali ruang hosting — untuk yang ingin jual hosting.

* **Static site hosts / CDN-first**

  * Untuk situs statis (HTML/CSS), sering lebih murah dan cepat (contoh: layanan hosting statis + CDN).

---

## 4. Komponen teknis yang sering digunakan

* **A Record** — arahkan domain/subdomain ke alamat IPv4 server.

  * Contoh: `example.com A 192.0.2.1`
* **AAAA Record** — seperti A tetapi untuk IPv6.
* **CNAME Record** — alias satu nama domain ke nama domain lain (tidak boleh untuk root domain).

  * Contoh: `www CNAME example.com`
* **MX Record** — mengatur server email untuk domain (prioritas disertakan).
* **TXT Record** — menaruh teks, sering dipakai untuk verifikasi (Google, Let's Encrypt, SPF).
* **NS Record** — menunjukkan nameserver untuk domain.
* **SPF / DKIM / DMARC** — catatan untuk validasi email, mengurangi spam & spoofing.
* **SSL/TLS Certificate** — enkripsi HTTPS; bisa gratis (Let's Encrypt) atau berbayar (Extended Validation, dsb).

---

## 5. Cara memilih domain & hosting (praktis)

Domain:

* Pilih nama singkat, mudah diingat/ketik, relevan, hindari tanda hubung bila bisa.
* Pilih ekstensi sesuai tujuan: `.com` umum, `.id` untuk lokal Indonesia, `.org` untuk organisasi, dsb.
* Periksa ketersediaan & harga renewal (kadang promo tahun pertama murah, renewal mahal).

Hosting:

* Pertimbangkan: estimasi traffic, jenis situs (statis/dinamis), kebutuhan database, budget, kemudahan penggunaan.
* Jika baru: shared hosting dengan SSL otomatis dan cPanel bisa cukup.
* Untuk toko online: prioritaskan SSL, backup otomatis, performa, dan opsi scaling.
* Baca SLA (uptime), dukungan teknis, lokasi data center (dekat pengunjung = latency lebih rendah).

---

## 6. Langkah praktis: dari beli domain sampai live

1. **Riset & beli domain** di registrar.
2. **Beli paket hosting** sesuai kebutuhan.
3. **Atur nameserver** di panel registrar → arahkan ke nameserver hosting (misal `ns1.host.com` & `ns2.host.com`).
4. Setelah DNS terpropagasi (bisa beberapa menit sampai 24–48 jam), **upload file situs** ke server:

   * Via FTP/SFTP (gunakan SFTP bila tersedia untuk keamanan) atau file manager di cPanel/Plesk.
5. **Buat database** (MySQL/MariaDB) bila situs dinamis (WordPress, Laravel, dsb).
6. **Instal CMS / deploy aplikasi** dan konfigurasi database.
7. **Pasang SSL/TLS** (Let's Encrypt atau sertifikat berbayar).
8. **Konfigurasi email** (bila pakai hosting yang mendukung email) atau gunakan layanan email pihak ketiga (G Suite/Outlook365).
9. **Test** akses, SSL, load time, email send/receive, form submission, mobile responsiveness.
10. **Backup** konfigurasi & data; atur jadwal backup.

---

## 7. Menghubungkan domain ke hosting: contoh record yang umum

* Root domain (`example.com`) → `A` → `192.0.2.1`
* `www.example.com` → `CNAME` → `example.com`
* Mail → `MX` → `mail.example.com` (priority 10) ; lalu `mail.example.com` → `A` → IP mail server
* Verifikasi & anti-spam: `TXT` → `v=spf1 include:mailprovider.com ~all`
* SSL challenge (Let's Encrypt) kadang pakai `TXT` atau http challenge otomatis.

---

## 8. Keamanan & best practices

* **Gunakan SFTP, bukan FTP** (SFTP enkripsi).
* **Instal SSL/TLS**; redirect HTTP → HTTPS.
* **Backup rutin** (file + database), simpan di lokasi terpisah.
* **Perbarui software** (CMS, plugin, server packages).
* **Batasi akses admin** (IP whitelisting jika perlu), gunakan password kuat & 2FA.
* **Set up WAF / rate limiting** untuk mencegah serangan.
* **Monitor uptime & error logs** (tools monitoring sederhana atau yang disediakan host).

---

## 9. Performa & optimisasi

* **Gunakan CDN** untuk mempercepat pengiriman aset statis ke pengguna global.
* **Caching**: server-side caching (Redis/opcache), plugin caching (WordPress), dan cache-control headers.
* **Optimalkan gambar & asset** (kompresi, WebP).
* **Gunakan HTTP/2 atau HTTP/3** bila didukung hosting untuk paralelisme transfer lebih baik.
* **Pilih lokasi server** yang dekat audiens utama.

---

## 10. Harga & model biaya (singkat)

* **Domain**: biaya tahunan; harga bervariasi per TLD, ada biaya registrasi & renewal.
* **Hosting**: model bulanan/tahunan; shared murah, VPS/dedicated/cloud lebih mahal.
* **Tambahan**: backup premium, sertifikat SSL berbayar, dedicated IP, layanan managed, CDN berbayar, layanan email enterprise.

---

## 11. Kesalahan umum pemula (hindari)

* Memilih nama domain panjang/susah diketik.
* Tidak memeriksa biaya renewal setelah tahun pertama.
* Tidak mengatur backup atau hanya mengandalkan host (backup harus dimiliki pemilik situs).
* Menggunakan FTP (plain) tanpa enkripsi.
* Mengabaikan SSL atau pemasangan yang salah (mixed content).
* Mengandalkan satu orang untuk akses admin tanpa recovery plan.

---

## 12. Checklist singkat sebelum go-live

* [ ] Domain aktif & nameserver mengarah benar
* [ ] Files & DB ter-upload & konfigurasi environment benar
* [ ] SSL terpasang & redirect HTTPS aktif
* [ ] Email (MX/SPF/DKIM) dikonfigurasi (jika perlu)
* [ ] Backup otomatis diatur
* [ ] Testing: form, login, pembayaran (jika ada)
* [ ] Performance: cek loading time, mobile friendly
* [ ] Security: update, SFTP, limit login, firewall dasar

---

## 13. Contoh skenario cepat

1. Buat situs WordPress sederhana (untuk pemula):

   * Beli domain & shared hosting dengan softaculous/one-click WordPress.
   * Gunakan nameserver hosting, install WordPress dari cPanel → pilih SSL Let's Encrypt → aktifkan caching plugin → setup email sender (SMTP plugin).

2. Deploy aplikasi Laravel (sedang berkembang):

   * Gunakan VPS (lebih fleksibel) atau managed cloud.
   * Upload code via Git, composer install, set `.env`, migrasi DB, konfigurasi queue & scheduler, gunakan SFTP & setup Nginx/Apache, pasang SSL, setup supervisor untuk queue.

---

## 14. Glosarium singkat (quick reference)

* **DNS**: Sistem terjemahan domain → IP.
* **Nameserver**: Server yang menyimpan DNS zone.
* **A/CNAME/MX/TXT**: Jenis record DNS.
* **Registrar**: Tempat membeli domain.
* **cPanel/Plesk**: Control panel hosting populer.
* **FTP/SFTP**: Protokol transfer file.
* **SSL/TLS**: Sertifikat enkripsi untuk HTTPS.
* **CDN**: Jaringan distribusi konten (cache global).
